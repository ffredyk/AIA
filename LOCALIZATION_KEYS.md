## Implementation Notes

### How to Use This Table:

1. **For English (en-US.resx)**:
   - Add each key with the English value
   - Example: 
     ```xml
     <data name="Tasks_NewTask" xml:space="preserve">
       <value>New Task</value>
     </data>
     ```

2. **For Czech (cs-CZ.resx)**:
   - Add each key with the Czech value
   - Example:
     ```xml
     <data name="Tasks_NewTask" xml:space="preserve">
       <value>Nový úkol</value>
     </data>
     ```

3. **Usage in XAML**:
   - All keys are used with the `{local:Localize}` markup extension
   - Example: `Text="{local:Localize Tasks_NewTask}"`

### Key Naming Convention:

- **Format**: `Category_SpecificItem`
- **Categories**: Tasks, TaskStatus, TaskPriority, RecurrenceType, RecurrenceUnit, Common
- **Benefits**: 
  - Easy to find related keys
  - Prevents naming conflicts
  - Clear context for translators

### Optional Keys:

Some keys may need plural forms for different languages. Consider adding:

- `Tasks_SelectedCount_Singular` = "1 task selected"
- `Tasks_SelectedCount_Plural` = "{0} tasks selected"

### Tooltip and Help Text:

For better UX, you might want to add tooltip variants:

- `Tasks_Duplicate_Tooltip` = "Create a copy of this task with all its subtasks"
- `Tasks_Archive_Tooltip` = "Hide this task from the main list but keep it in storage"
